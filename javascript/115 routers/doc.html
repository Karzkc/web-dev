React Router Documentation

Core Concepts

React Router enables client-side routing in React applications. It helps manage navigation and URL changes without full page reloads.

Key Components

1. createBrowserRouter

Creates a router configuration for web browsers. Handles dynamic client-side routing.

import { createBrowserRouter } from 'react-router-dom';

const router = createBrowserRouter([
  {
    path: "/",
    element: <HomePage />
  },
  {
    path: "/about",
    element: <AboutPage />
  }
]);

2. RouterProvider

Makes the router available to your entire application:

import { RouterProvider } from 'react-router-dom';

function App() {
  return <RouterProvider router={router} />;
}

3. Link & NavLink

Navigation components that prevent full page reloads:

import { Link, NavLink } from 'react-router-dom';

// Basic Link
<Link to="/about">About</Link>

// NavLink with active styling
<NavLink 
  to="/about" 
  style={({ isActive }) => ({
    color: isActive ? 'blue' : 'black'
  })}
>
  About
</NavLink>
Key Differences:
  • Link: Basic navigation link
  • NavLink: Adds automatic active state tracking

Route Configuration

Route objects contain:

  • path: URL path pattern
  • element: Component to render
  • children: Nested routes

Dynamic Routing Example

{
  path: "/user/:username",
  element: <UserProfile />
}

Access parameters with useParams():

import { useParams } from 'react-router-dom';

function UserProfile() {
  const { username } = useParams();
  return <h1>Profile: {username}</h1>;
}

Navigation Props

to prop usage examples:

// Absolute path
<Link to="/about">About</Link>

// Relative path
<Link to="../parent">Go Up</Link>

// With parameters
<Link to="/user/123">User Profile</Link>

// With query parameters
<Link to="/products?sort=price">Products</Link>

Best Practices

  • Use createBrowserRouter for web applications
  • Wrap root component with RouterProvider
  • Use NavLink for navigation menus
  • Organize routes in a central configuration
  • Use lazy loading for large components
Remember:
  • Router components must be used within a RouterProvider
  • Always define a root route (path: "/")
  • Use error boundaries for error handling
