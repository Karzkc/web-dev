JavaScript Error Handling Documentation

1. Introduction to Error Handling

Error handling in JavaScript ensures your code doesn't break unexpectedly. JavaScript uses try-catch blocks to handle errors gracefully. By using these blocks, you can manage unexpected situations and avoid app crashes.

Let’s explore the most common techniques:

  • try-catch block – to catch and handle runtime errors.
  • finally block – runs code regardless of success or failure.
  • throw statement – allows you to create custom errors.

2. Example: Throwing an Error for Invalid Input

Let's start by catching input errors, like non-numeric inputs:

let a = prompt('num1: ');
let b = prompt('num2: ');

if (isNaN(a) || isNaN(b)) {
    throw new SyntaxError("Invalid input: Please enter a valid number.");
}

let c = parseInt(a) + parseInt(b);
console.log("Result:", c);

This code prompts the user for two numbers. If the input is not a number, it throws a custom error using the throw statement.

3. Handling Errors with try-catch

The try-catch block allows you to handle errors in a controlled way. If an error occurs in the try block, it’s caught in the catch block.

try {
    let d = c + x; // 'x' is not defined
    console.log(d);
} catch (error) {
    console.log("Error occurred:", error.message);
} finally {
    console.log("This always runs, regardless of the error.");
}

The finally block always executes, whether or not an error occurred. It’s useful for cleanup tasks or final actions that must run in any scenario.

4. Overview of try-catch-finally

Here's how JavaScript handles error scenarios:

  • try: Tests the block of code for errors.
  • catch: Executes when an error is found in the try block.
  • finally: Always executes after try-catch, regardless of the outcome.

5. When to Use throw

The throw statement is used when you want to manually trigger an error. For example, when the input doesn’t meet your requirements.

6. Conclusion

By using try, catch, finally, and throw, JavaScript provides robust mechanisms for managing errors and preventing your application from crashing due to unexpected inputs or runtime issues.
