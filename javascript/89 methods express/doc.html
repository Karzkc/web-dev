ExpressJS Documentation

1. Introduction to ExpressJS Methods

ExpressJS provides a variety of HTTP methods to interact with the server:

  • GET - Retrieves data from the server.
  • POST - Sends data to the server to create a new resource.
  • PUT - Updates an existing resource on the server.
  • DELETE - Deletes a resource on the server.
  • LISTEN - Specifies the port on which the server should listen.
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => res.send('GET request example'));
app.post('/', (req, res) => res.send('POST request example'));
app.put('/', (req, res) => res.send('PUT request example'));
app.listen(port, () => console.log(`Server running on port ${port}`));

2. Setting Up Static Files with Public Folder

Using express.static, you can serve static files such as HTML, CSS, and JS files. In this example, all files inside the public folder will be accessible publicly.

app.use(express.static('path/to/public'));

Example file structure:

project-folder/
    public/
        index.html
        style.css
        script.js

3. Routing in ExpressJS

Express provides easy-to-use routing. Here are some examples:

app.get('/path', (req, res) => res.send('GET response'));
app.post('/path', (req, res) => res.send('POST response'));

// Parameterized routes
app.get('/user/:id', (req, res) => res.send(`User ID: ${req.params.id}`));

Example of routing in modules:

const blog = require('./routes/blog');
app.use('/blog', blog);

4. Interconnecting Files and Templates

To serve HTML files, use res.sendFile:

app.get('/index', (req, res) => {
    res.sendFile('/templates/index.html', { root: __dirname });
});

This allows you to send specific HTML files based on routes.

5. Using JSON Responses

Express allows easy JSON responses:

app.get('/api', (req, res) => {
    res.json({
        name: "Kartik",
        version: "1.0.0",
        description: "Amazing package"
    });
});

Using Postman for HTTPS Servers and Requests

Postman is a powerful tool that allows developers to test, document, and manage APIs efficiently. It provides a user-friendly interface for creating and managing workspaces, which can be used to handle HTTPS servers and make various types of requests.

Getting Started with Postman

To start using Postman, download and install the application from the official Postman website. Once installed, you can create a new workspace or import an existing one.

Creating a Workspace

Workspaces in Postman allow you to group related requests, environments, and other resources together. To create a workspace:

  1. Open Postman and click on the "Workspaces" tab.
  2. Click the "Create Workspace" button.
  3. Provide a name and description for your workspace, and configure the visibility and other settings as needed.
  4. Click "Create Workspace" to finalize.

Making HTTPS Requests

Postman simplifies the process of making HTTPS requests. Follow these steps to make your first request:

  1. Open your workspace and click on the "New" button.
  2. Select "HTTP Request" from the menu.
  3. In the new request tab, enter the URL of the server you want to request data from.
  4. Choose the request method (GET, POST, PUT, DELETE, etc.) from the dropdown menu.
  5. Add any necessary headers, body parameters, and query parameters.
  6. Click the "Send" button to make the request.

The response from the server will be displayed in the lower pane, providing detailed information about the status, headers, and body of the response.

Using Collections and Environments

Postman allows you to organize requests into collections and use environments to manage different variables for development, staging, and production.

  • Collections: Group related requests together for easy access and management.
  • Environments: Define environments with different sets of variables (e.g., base URLs, authentication tokens) for seamless switching between different stages of development.

Conclusion

Postman is an essential tool for developers working with APIs, providing powerful features for testing, documenting, and managing API requests and responses. By leveraging workspaces, collections, and environments, you can efficiently handle complex scenarios and ensure your API workflows are smooth and error-free.
