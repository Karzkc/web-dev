ExpressJS Documentation

Express is a minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications. This documentation covers the basics of setting up an Express server, handling routes, and utilizing static files through the public folder.

Getting Started with Express

To use Express, install it via npm:

npm install express

Basic Server Setup

In Express, you can create a server with just a few lines of code. Here's a basic setup:

const express = require('express');
const app = express();
const port = 3000;

app.listen(port, () => {
console.log(`Server running at http://localhost:${port}/`);
});

Serving Static Files with the Public Folder

Express allows you to serve static files, such as HTML, CSS, and JavaScript, using the express.static middleware. You can place these files in a public folder, making them accessible to the client.

To serve files from a directory, use:

app.use(express.static('path/to/public/folder'));

In the code example:

app.use(express.static('d:\\Kartik Khiriya\\own\\new\\web dev\\javascript\\88 express\\public'));

Now, any files in the public directory, such as a text file, image, or script, are accessible to anyone who visits your server's URL.

Routing in Express

Express provides an easy way to handle different routes in your application. In the setup below, a basic route handler listens on the / route:

app.get('/', (req, res) => {
res.send('Hello World! how are you');
});

Dynamic Routes with URL Parameters

Dynamic routes allow you to capture values from the URL. In the code example:

app.get('/:slug/:second', (req, res) => {
res.send(`hello this is ${req.params.slug} and ${req.params.second}`);
console.log(req.params); // { slug: 'value', second: 'value' }
console.log(req.query);
});

This route captures the values passed as slug and second in the URL, enabling you to handle specific values dynamically.

Additional Features in Express

  • Query Parameters: req.query allows access to query strings in the URL (e.g., ?dark=true).
  • Middleware: Middleware functions like express.static can handle file serving, logging, authentication, etc., within your application.
  • Error Handling: You can add custom error-handling middleware to manage errors and exceptions more effectively.

Starting the Server

To start your server, run:

node filename.js

The server will be accessible at http://localhost:3000.

Conclusion

This covers the basics of ExpressJS, including serving static files, routing, and utilizing the public folder. Express allows efficient handling of server-side routes and makes it easy to create dynamic and responsive web applications.
