JavaScript Events Documentation

This document provides a detailed overview of important JavaScript event listeners, event bubbling, and timer functions, including setInterval and setTimeout. Each section includes definitions, examples, and expected outputs.

Event Listeners

Event listeners are functions that run in response to specific events occurring on an element (such as a click, keyboard input, etc.). Below are some commonly used event listeners:

1. Click Event Listener

The click event listener triggers when the user clicks on the element.

document.getElementById('myButton').addEventListener('click', function() {
    alert('Button clicked!');
});
Example: Clicking a button triggers an alert box that says "Button clicked!".

2. Context Menu Event Listener

The contextmenu event triggers when the user right-clicks on an element, opening the context menu.

document.getElementById('myDiv').addEventListener('contextmenu', function(e) {
    e.preventDefault();  // Prevents the default right-click menu
    alert('Right-click detected!');
});
Example: Right-clicking on a specified element displays an alert and prevents the browser’s default context menu.

3. Keydown Event Listener

The keydown event fires whenever the user presses a key on the keyboard. You can access the key's value and its keyCode.

document.addEventListener('keydown', function(event) {
    alert('Key pressed: ' + event.key + ' (code: ' + event.keyCode + ')');
});
Example: When a key is pressed, an alert shows the key and its keyCode.

Event Bubbling

Event bubbling describes how an event propagates from the target element to its parent elements. It allows events to be captured at the element where they originated, and then bubble up to the document’s root.

Example of Event Bubbling:

The following example illustrates how click events propagate from a child element to its parents, but we stop the propagation using stopPropagation():

// Stopping the propagation at the child level
document.querySelector(".child").addEventListener("click", function(e) {
    e.stopPropagation();
    this.style.backgroundColor = "cyan";
});

document.querySelector(".papa").addEventListener("click", function(e) {
    this.style.backgroundColor = "pink";
});

document.querySelector(".dada").addEventListener("click", function(e) {
    this.style.backgroundColor = "green";
});
Example: Clicking on the child changes its background color without affecting the parent elements, thanks to stopPropagation().

Output:

  • Click on the child element: Only the child changes color (cyan).
  • Click on the parent element: The parent changes color (pink) and so does the grandparent (green), unless stopPropagation() is used.

Timers

Timers in JavaScript allow you to delay or repeat the execution of a function after a certain amount of time. The two main timer functions are setTimeout and setInterval.

1. setTimeout

setTimeout is used to execute a function after a specific delay, measured in milliseconds.

setTimeout(function() {
    console.log('Executed after 2 seconds');
}, 2000);
Example: Logs the message "Executed after 2 seconds" to the console after a 2-second delay.

2. setInterval

setInterval repeatedly calls a function at specified intervals (in milliseconds).

setInterval(function() {
    console.log('Repeating every second');
}, 1000);
Example: Logs "Repeating every second" to the console every second until clearInterval is called.

Clearing Timers

You can stop a timer using clearTimeout or clearInterval by storing the timer ID returned from setTimeout or setInterval.

let intervalId = setInterval(function() {
    console.log('Repeating');
}, 1000);

// To stop the interval after 5 seconds
setTimeout(function() {
    clearInterval(intervalId);
    console.log('Interval stopped');
}, 5000);
Example: The interval runs every second but is stopped after 5 seconds.
Note: Always use clearInterval or clearTimeout to stop timers when they are no longer needed to avoid memory leaks.
