position

The position property specifies the positioning of an element relative to its normal position. by default its static .

static -

we cant apply top bottom left right z index on static we have to change position to relative, absolute, fixed, sticky.


relative

The relative positioning of an element is unchanged by the positioning of other elements. It maintains its original position relative to where it would have been if no positioning properties had been applied.
we can apply top bottom left right z index on relative
with top bottom left right we can add position
relative mtlb apne purani pos se kitna dur


z index

The z-index property specifies the stack order of an element. An element with a higher value will appear in front of an element with a lower value. If two or more elements have the same stack order, their order is determined by the order they appear in the HTML.
isme jiski sbse jyada property wo utna upar


absolute

The absolute positioning of an element is related to its first positioned (not static) ancestor element. if no such ancestor, then it is positioned relative to the viewport.
isme apne papa agar relative hai to uske reference se pos vrna body and at last html - nearest position ancestor ke - if papa also absoulte then uske according

fixed -

The fixed positioning of an element is related to the viewport, meaning it stays in the same place even when scrolling. If no scrollbars are present, the fixed positioned element remains fixed relative to the viewport.
isme agar fixed kr diya to kuch bhi hojaye wahi pos rhegi even scroll kre to bhi

sticky

The sticky positioning of an element is relative to the nearest positioned ancestor, but it will stop at a specified container when scrolling.
if sticky element is in the top then it will stop at the top of the container when scrolling down and from top when scrolling up
