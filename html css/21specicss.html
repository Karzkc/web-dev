cascade algorithm -

CSS stands for Cascading Stylesheets. The cascade is the algorithm for solving conflicts where multiple CSS rules apply to an HTML element. It's the reason that the text of the button styled with the above CSS will be purple. Understanding the cascade algorithm helps you understand how the browser resolves conflicts like this. The cascade algorithm has 4 distinct stages. Position and order of appearance: the order in which your CSS rules appear Specificity: an algorithm that determines which CSS selector has the strongest match Origin: the order in which CSS appears and where it comes from, whether that is a browser style, CSS from a browser extension, or your authored CSS Importance: some CSS rules are weighted more heavily than others, especially with the !important rule type Lets look into all these one by one

Browser Style: - These are styles that are automatically applied by the browser. For example, the browser applies the default font, background color, and text color to all elements.

important!- - This rule makes a CSS rule take precedence over the browser's default styles. It's used to override its syntax is - <p style="color: red!important;"> </p>

Specificity Calculation To calculate specificity, assign a value to each part of the selector: 

                Universal Selector: 0
            Element selectors and pseudo-elements: 1
            Class selectors, attribute selectors, and pseudo-classes: 10
            ID selectors: 100
            Inline styles: 1000
